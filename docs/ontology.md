# Онтология AIST Track Bot

> **Этот документ является единственным источником истины для терминологии бота.**
> Все разработчики и ИИ-ассистенты должны использовать только термины из этого документа.

---

## 1. Участники

| Термин | Определение |
|--------|-------------|
| **Пользователь** | Любой человек, взаимодействующий с ботом |
| **Ученик** | Пользователь, проходящий Марафон |
| **Читатель** | Пользователь, использующий Ленту |

---

## 2. Режимы обучения

| Термин | Определение |
|--------|-------------|
| **Марафон** | Рассчитанная на 14-дневное персональное изучение программа по системному развитию |
| **Лента** | Адаптированные под интересы пользователя ежедневные дайджесты, которые изучаются в своём ритме за рамками Марафона |

---

## 3. Единицы обучения

| Термин | Режим | Определение | Состав |
|--------|-------|-------------|--------|
| **Урок** | Марафон | Единица теоретического обучения | Материал урока + Вопрос урока |
| **Задание** | Марафон | Единица практического обучения | Материал задания + Запрос ответа |
| **Дайджест** | Лента | Ежедневная порция материала | Материал дайджеста + Вопрос дайджеста |
| **Неделя** | Лента | Цикл планирования в Ленте | Меню тем → 5-7 дайджестов |

---

## 4. Типы сообщений

### Иерархия

```
СООБЩЕНИЕ
├── Материал (текст для изучения)
│   ├── Материал урока
│   ├── Материал задания
│   └── Дайджест
├── Вопрос (требует содержательного ответа)
│   ├── Вопрос урока
│   ├── Бонусный вопрос
│   ├── Вопрос дайджеста
│   └── Вопрос пользователя
├── Обратная связь (реакция на действие)
│   ├── Проверка
│   ├── Подсказка
│   └── Консультация
├── Запрос (организационный, обязателен для ответа)
│   ├── Запрос данных
│   └── Меню тем
├── Сводка (информационное)
│   ├── Профиль
│   └── Статистика
├── Служебное
│   ├── Подтверждение
│   └── Ошибка
└── Ответ (от пользователя)
    ├── Ответ на урок
    ├── Ответ на задание
    ├── Ответ на бонус
    └── Фиксация
```

### Таблица типов сообщений

| Основной тип | Конкретный вид | Отправитель | Режим | Описание |
|--------------|----------------|-------------|-------|----------|
| **Материал** | Материал урока | Бот | Марафон | Теоретический текст для изучения |
| **Материал** | Материал задания | Бот | Марафон | Текст инструкции к практике (создание РП) |
| **Материал** | Дайджест | Бот | Лента | Ежедневный текст для чтения |
| **Вопрос** | Вопрос урока | Бот | Марафон | Проверка понимания теории |
| **Вопрос** | Бонусный вопрос | Бот | Марафон | Дополнительный вопрос повышенной сложности |
| **Вопрос** | Вопрос дайджеста | Бот | Лента | Вопрос для фиксации |
| **Вопрос** | Вопрос пользователя | Пользователь | Оба | Вопрос в свободной форме |
| **Обратная связь** | Проверка | Бот | Оба | Оценка/комментарий к ответу пользователя |
| **Обратная связь** | Подсказка | Бот | Оба | Помощь при затруднении |
| **Обратная связь** | Консультация | Бот | Оба | Ответ на вопрос пользователя |
| **Запрос** | Запрос данных | Бот | Оба | Запрос имени, языка и т.п. |
| **Запрос** | Меню тем | Бот | Лента | Предложение тем на выбор |
| **Сводка** | Профиль | Бот | Оба | Информация о пользователе |
| **Сводка** | Статистика | Бот | Оба | Прогресс, достижения |
| **Служебное** | Подтверждение | Бот | Оба | "Принято", "Генерирую..." |
| **Служебное** | Ошибка | Бот | Оба | Сообщение об ошибке |
| **Ответ** | Ответ на урок | Пользователь | Марафон | Ответ на вопрос теории |
| **Ответ** | Ответ на задание | Пользователь | Марафон | Описание рабочего продукта |
| **Ответ** | Ответ на бонус | Пользователь | Марафон | Ответ на бонусный вопрос |
| **Ответ** | Фиксация | Пользователь | Лента | Личный вывод после дайджеста |

## 5. Связанные понятия

| Термин | Определение |
|--------|-------------|
| **Тема** | Область знаний для изучения (применимо к Марафону и Ленте) |
| **Рабочий продукт (РП)** | Результат выполнения Задания — артефакт, который описывает Ученик в Ответе на задание |
| **День** | Порядковый номер в Марафоне (День 1, День 2...) или календарная дата в Ленте |
| **Сложность** | Когнитивный уровень вопросов (1 — различение, 2 — понимание, 3 — применение) |

---

## 6. Статусы

| Сущность | Статусы |
|----------|---------|
| **Марафон** | `not_started` → `active` → `paused` / `completed` |
| **Лента** | `not_started` → `active` → `paused` |
| **Неделя (Лента)** | `planning` → `active` → `completed` |
| **Урок/Задание** | `pending` → `in_progress` → `completed` |

---

## 7. Соответствие терминов и кода

| Термин онтологии | Текущее в коде | Целевое в коде |
|------------------|----------------|----------------|
| Урок | `theory` | `lesson` |
| Материал урока | `theory content` | `lesson_text` |
| Вопрос урока | `question` | `lesson_question` |
| Задание | `practice` | `task` |
| Материал задания | `practice content` | `task_text` |
| Дайджест | `feed_session`, `content` | `digest` |
| Вопрос дайджеста | `reflection_prompt` | `digest_question` |
| Фиксация | `fixation`, `fixation_text` | `fixation` |
| Меню тем | `suggested_topics` | `topic_menu` |
| Ответ на урок | `theory_answer` | `lesson_answer` |
| Ответ на задание | `work_product` | `task_answer` |
| Сложность | `bloom_level`, `complexity_level` | `complexity` |
| Вопрос пользователя | `IntentType.QUESTION` | `user_question` |
| Консультация | `handle_question` response | `consultation` |

---

## 8. Правила использования терминологии

1. **Всегда использовать термины из этого документа** в коде, комментариях, сообщениях пользователю и документации.

2. **При неясности** — уточнять у владельца продукта, а не придумывать новые термины.

3. **При добавлении новых сущностей** — сначала добавить в этот документ, затем в код.

4. **Язык терминов**:
   - В коде: английские эквиваленты (см. таблицу соответствия)
   - В интерфейсе: русские термины
   - В документации: русские термины с английскими в скобках при первом упоминании
